# Document Style Guide

## Purpose

Maintain consistent, professional, and actionable documentation quality.

## Writing Rules

- Use concise, direct language.
- Prefer numbered procedures for execution steps.
- Define acronyms on first use.
- Avoid ambiguous terms like "soon", "maybe", or "as needed" without criteria.

## Procedure Format

Each operational procedure should include:

1. Purpose
2. Preconditions
3. Steps
4. Validation
5. Rollback or recovery path

## Versioning

- Add a "Last updated" date to major documents.
- Update `CHANGELOG.md` for material process changes.
