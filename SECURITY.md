# Security Policy

## Reporting a Vulnerability

If you discover a security issue related to this repository's automation, scripts, or operational guidance:

1. Do not open a public issue.
2. Report privately via GitHub Security Advisories for this repository.
3. Include reproduction steps, impact, and suggested mitigation if available.

## Response Targets

- Initial acknowledgment: within 72 hours
- Triage decision: within 7 days
- Resolution timeline: based on severity and operational risk

## Scope

In-scope concerns include:

- Credential leakage in documentation or examples
- Insecure defaults in automation assets
- Privilege escalation opportunities in operational procedures

Out-of-scope concerns include:

- General best-practice disagreements without demonstrated risk
- Third-party platform vulnerabilities unrelated to repository content
